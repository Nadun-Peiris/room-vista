# PUSL3122 Submission Compliance Checklist (Room Vista)

Use this checklist before final submission.

## 1) Deliverables

- [ ] Single PDF report submitted on DLE (no external folder-only submission)
- [ ] Report is ~2000 words (marker may stop reading after 2000)
- [ ] GitHub repository link included in report and publicly accessible
- [ ] Video link included in report and publicly accessible
- [ ] Video duration is 7-12 minutes
- [ ] All group members appear clearly in the video
- [ ] Video demonstrates software running with data

## 2) Report Structure (Required Headings)

- [ ] Introduction
- [ ] Background
- [ ] Gathering Data
- [ ] Design
- [ ] Implementation
- [ ] Evaluation
- [ ] Summary
- [ ] References

## 3) HCI / UX Process Evidence

- [ ] Context of use defined (who uses the system and in what scenario)
- [ ] Additional requirements gathered beyond brief requirements
- [ ] Methods/techniques for data gathering documented
- [ ] Personas included
- [ ] User stories included
- [ ] Storyboard(s) included
- [ ] Low-fidelity prototype included
- [ ] High-fidelity prototype included
- [ ] Iteration from low-fi feedback to high-fi shown

## 4) User Testing Evidence

- [ ] Formative study completed
- [ ] Summative study completed
- [ ] At least 2 potential users in each study
- [ ] Participant demographics included (no identity disclosure)
- [ ] Consent process/form documented
- [ ] Test plan provided
- [ ] Method/technique justified
- [ ] Study setup evidence included (screenshots/photos where appropriate)
- [ ] Findings summarized
- [ ] Recommendations and resulting changes documented

## 5) Appendix A Functional Requirements Coverage

- [ ] Designers can log in
- [ ] Enter/store room specs (size, shape, color scheme)
- [ ] Create and visualize designs in 2D
- [ ] Convert 2D design to 3D view
- [ ] Furniture scaling fits room dimensions
- [ ] Shading controls for room/furniture
- [ ] Color controls for room/furniture
- [ ] Save completed designs
- [ ] Edit existing designs
- [ ] Delete existing designs

## 6) Non-Functional Requirements Evidence

- [ ] Usability decisions explained (intuitive, consistency, learnability)
- [ ] Accessibility considerations documented (contrast, readability, navigation)
- [ ] Performance behavior reported (2D/3D rendering responsiveness)
- [ ] Action feedback demonstrated (save/update/errors, etc.)
- [ ] Error prevention/recovery demonstrated (prompts/undo)
- [ ] Efficiency explained (minimal steps for key tasks)
- [ ] Engagement explained (interactive/immersive 3D features)

## 7) Git Repository Requirements

- [ ] README is project-specific and complete
- [ ] README credits all external assets/resources
- [ ] No `.zip` or compressed previous versions committed
- [ ] Commit messages are meaningful
- [ ] Commit history shows consistent progress over time

## 8) Final Validation Pass

- [ ] Run `npm run lint` successfully
- [ ] Run app and test full user flow: login -> dashboard -> editor -> save/load
- [ ] Test both superadmin and designer roles
- [ ] Test furniture upload/create/update/delete
- [ ] Re-check all links in report (GitHub + video)
- [ ] Spell-check and reference-check report
